# https 환경에서 http 이미지를 호출하면 안되는 이유

---

로컬 리액트 환경(http)에서 api로부터 받아온 imgUrl을 src태그에 넣었더니,
이미지가 잘 불러와졌다.

그런데 운영환경(https)으로 환경을 바꾸니 특정 이미지들이 안불러와져서
로그부터 뜯어보니

`Failed to load resource: net::ERR_CERT_COMMON_NAME_INVALID `

이렇게 뜨는 점을 확인하였다.

해당 오류는

'크롬에서 신뢰할 수 있는 서버 인증서를 사용하지 않은 경우 나타나는 오류'라고 한다.

즉 HTTPS(보안 소켓 계층)로 보호되는 사이트에서 HTTP(보안 소켓 계층 없음)로 제공되는 이미지를 가져와서 생긴
브라우저 보안 정책 이슈라는 것.

해결책은 https 이미지를 사용하는 것이다.
